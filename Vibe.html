Vibe
This was my midway capstone project for a Front-End Design and Development Bootcamp. In this project, I applied both my user experience design and front-end development skills. Vibe is an application which simplifies the process of creating a collaborative playlist.
PROBLEM STATEMENT
Develop an efficient way to create collaborative playlists at social gatherings
COMPETITIVE ANALYSIS
Existing Collaborative Playlist Applications Either...
Require Spotify Premium
Spotify Premium is the main music source where users are able to create and listen to their custom playlists at any point in time. All outside source collaborative playlist applications require a Spotify Premium account.
Have a Complex Flow
Existing methods of creating collaborative playlists require all users to share the same music source. They also require a complex process, such as text message or emailing the playlist from user to user.

Interview Insights
ACCESSIBLE
It is important the music source is easily accessible to everyone. Ideally, the source will not require a membership and be unpaid.
COLLABORATIVE
Listening to music can be a social experience. The app should help enhance social experiences and encourage participation.
CONTROL
With multiple users participating, it is important there are limits, for example, bad song requests, too many song requests etc.
INTUITIVE
The app should be simple & convenient. Idea is not to replace the user's music source, but be a tool for users when they want to create collaborative playlists.

Vibe
HOST OR JOIN PLAYLISTS
Users are able to either create a new playlist for others to join or join other user's playlists
PLAYLIST CODE
When creating a playlist, the user sets a five digit code. Users joining playlists must enter the matching code to get access to the playlist.
ADDING SONGS
Users are able to search YouTube to add songs to their playlist. When a user creates a playlist, they are able to view pre-existing playlists. Users are able to add an entire playlist to get the music started.
PLAYLIST VIEW
Options when viewing the playlist depend on whether the user hosted the playlist or joined the playlist. If the user is the host, they have complete control. Hosts are able to play, skip, shuffle, add, reorder and delete songs. If the user joined the playlist they are only able to add songs and delete only the songs they have added to the playlist.
HOMEPAGE
When an existing user opens the app, they view all of their playlists, both hosted and joined. They also have easy access to creating or joining a new playlist by toggling to New Playlists.
